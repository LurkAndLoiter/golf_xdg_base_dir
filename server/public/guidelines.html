Challenge Submission Guidelines

Learn how to create high-quality VimGolf challenges that others will enjoy solving.

General Principles

  • ✓ Make your challenge focused on a specific Vim technique or concept
  • ✓ Ensure there's a clear and logical path from start text to target text
  • ✓ Test your solution thoroughly before submitting
  • ✗ Avoid challenges that can be solved with a single substitution command

Writing Good Descriptions

  • ✓ Clearly explain what needs to be done
  • ✓ Include any specific requirements or constraints
  • ✓ Mention if the challenge focuses on specific Vim features

Setting the Par Score

  • ✓ Count the minimum keystrokes needed for your solution
  • ✓ Add a small buffer (2-3 keystrokes) to account for alternative solutions
  • ✗ Don't make the par unreasonably low or high

Choosing Tags

  • ✓ Include relevant Vim commands (e.g., substitute, macro, visual)
  • ✓ Add tags for specific techniques (e.g., regex, motion, text-object)
  • ✗ Avoid generic or irrelevant tags
